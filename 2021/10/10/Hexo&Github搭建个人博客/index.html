Hexo&Github建立个人博客(零基础)

发表于2021-10-10|更新于2021-10-10|Blogs
|阅读量:

前言

监守者的唠叨

  相比现有的博客写作平台，自我搭建并坚持完善更新很难，毕竟就地摆摊和精装创业难度相差不是随便可以忽视的。但是完成一件从无到有的事是很有成就感的!
  鉴于面向小白(监守者自身实力有限)，此次仅是实现配置安装并部署博客的简要记录，涉及知识深度和广度并不深。
  衷心祝愿有一个好的结果，有始有终。作为一个毫无基础的小白，七天的跌跌撞撞才堪堪完善。Bug的排查和修复就需要坐的住，跟何况是没有领路人的小白。这很磨人耐心，望勤勉成功！

监守者的警示

  生活的快节奏使得一切行动裹挟着目的与功利，这无可厚非，但急迫往往会适得其反。
  工具与技术更新迭代很快，此次记录不代表永远一尘不变，毕竟科学仅是当下人类最有效的总结罢了。
  记得注意使用的工具以及版本号，和博客更新的日期！教程往往也有一定的局限和时效，当然监守者会努力的维持遗思之地的运转与清扫。

  原理：使用GitHub提供的GitHub Pages（免费静态站点托管服务），在GitHub仓库中托管和发布自己的静态网站页面
  优点：完全免费；静态站点；轻量快速；自由定制；迁移方便…
  缺点：发文不便；依赖本地；Github国内访问速度受限
  提示：
      ·txt文本直接操作只能撤销一步
      ·改动前尤其改动源码，备份能省去好多麻烦
      ·主题文档一般更全、更系统，故拓展前一定先看操作文档
      ·版本很重要，涉及到兼容问题；软件主题使用人越多意味着教程更新更多
      ·关注更新时间和人群，跟着比较全的教程，新手忌新的旧的、各式各样的教程混改

预备配置

  提示：若打不开或下载缓慢可能由于国内网络下载受限，需要科学上网

Node.js

  1、Node.js是一个基于 Chrome V8 引擎的JavaScript 运行环境；使用一个事件驱动、非阻塞式I/O的模型；是一个运行环境，或者说是一个JS语言解释器。
  2、下载地址：Node.js_64(位)；检验并查看版本号：node -v & npm -v （node和npm一起）

Git

  1、Git是以一个开源分布式版本控制系统
  2、下载地址：Git-2.11.0.3-64-bit_windows；检验并查看版本号：git --version

Hexo

  1、Hexo是一个博客框架，使用Markdown(或其他渲染引擎)解析文章
  2、下载方式：利用Node.js环境，在控制台输入：npm install hexo-cli -g；检验并查看版本号：Hexo -v

部分指令释意

npm

  1、install：会有可能遇到简写成 i
  2、-g：安装到全局；–save-dev：安装到开发环境；–save：安装到生产环境（npm i -S xxx）
  3、npm uninstall xxx：删除xxx模块

Hexo

  1、Hexo init：当前文件夹下创建初始化的目录文件
  2、hexo generate(hexo g)：生成静态文件
  3、hexo server(hexo s)：启动服务器，默认访问网址：http://localhost:4000/
  4、hexo deploy(hexo d)：部署网站，可以理解为将之静态文件安置到网上

补充

  1、记录所使用配置：·系统win10 ·node -v14.17.6； ·npm -v6.14.15； ·git version 2.33.0.windows.2； ·hexo -5.3.0
  2、现阶段Github部署branch已由master改为main

搭建仓库

Github相关

  1、注册拥有一个属于自己的GitHub账号，推荐使用国外邮箱（可能后期插件开发等问题受限）或其他邮箱注册
  2、利用Github仓库作为Hexo博客的服务器；
    点击顺序：右上角个人图像 → Your repositories → New → 按规定填写注意命名规则 → Create repository

图片


绑定密钥(SSH)

  1、本地进入任意文件夹内 → 右击选择Git Bash Here进入 → 输入ssh(验证是否装入ssh) → ssh-keygen -t rsa -C 注册的邮件账号(生成ssh) → 等待执行并相继点击四次回车 → C盘/Users(用户)/管理员文件夹/.ssh文件夹 → .ssh文件夹内有两个文件：id_rsa和id_rsa.pub

图片

  2、打开id_rsa.pub文件并复制里面全部代码 → 打开GitHub里Settings → 右侧栏SSH and GPG keys →New SSH key → Title名字随意 → 粘贴刚刚复制的那串代码于Key文本框内 → Add SSH key → Hexo命令框内：“ssh -T git@github.com 中途输入yes”（测试ssh是否绑定成功）

本地生成博客

**  1、新建一个空文件夹 → 进入 Git Bush Here → 输入 “hexo init” (前提以安装完Hexo) → 输入 “hexo d” → 输入 “hexo s” →本地浏览器 “http://localhost:4000” 进入可预览

报错

  扩展：可以先初步尝试整体更换主题：Hexo_theme官网、Butterfly主题、NexT主题、Melody主题……
      下载后在 “hexo init” 的文件夹(博客根目录)内，themes文件夹内会存在新下的主题文件夹，将博客根目录内 _config.yml文件 打开，找到#Extensions ## Plugins: https:…theme:loadings，将loadings更改为所更换主题的名称；主题文件夹和博客根文件夹许多名称相似是因为合并编译的需要，后期切记不要弄混文件夹
